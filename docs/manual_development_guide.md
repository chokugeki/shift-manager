# AIなしでの個人開発ガイド（指示書）

本ドキュメントは、AIアシスタント（ChatGPT, Copilot等）を使用せずに、個人が独力で「チャート式シフト管理システム」と同等のアプリケーションを開発するための学習リソースおよび作業フローをまとめたものです。

## 1. 必要なリソースと学習媒体

開発に必要な知識を獲得するための信頼できる情報源です。AIに頼らない場合、一次情報（公式ドキュメント）と体系的な学習教材が重要になります。

### 1.1 必須技術スタックと学習リソース
| 技術分野 | 推奨リソース (Web) | 推奨書籍 (例) |
| :--- | :--- | :--- |
| **HTML/CSS/JS** | [MDN Web Docs](https://developer.mozilla.org/ja/) (Mozilla) | 「プロのWebサイト制作の教科書」シリーズなど |
| **React** | [React 公式ドキュメント](https://ja.react.dev/) (特に「クイックスタート」) | 「モダンJavaScriptの基本から始める React実践の教科書」 |
| **Next.js** | [Next.js Documentation](https://nextjs.org/docs) (英語/一部日本語訳あり) | 「Next.js実践ガイド」等の最新技術書 |
| **TypeScript** | [TypeScript Handbook](https://www.typescriptlang.org/docs/) | 「プロを目指す人のためのTypeScript入門」 |
| **Tailwind CSS** | [Tailwind CSS Docs](https://tailwindcss.com/docs) | (公式ドキュメントが非常に充実しているため推奨) |

### 1.2 困ったときの解決リソース
- **Stack Overflow**: エラー解決策の検索。
- **Qiita / Zenn**: 日本語での技術記事検索（ただし情報の鮮度に注意）。
- **GitHub Issues**: ライブラリのバグや仕様確認。

## 2. 開発ワークフロー

以下の手順で段階的に開発を進めます。

### フェーズ1: 基礎学習と環境構築 (所要目安: 1〜2週間)
1.  **開発環境の準備**:
    -   Node.js (LTS版) のインストール。
    -   コードエディタ (VS Code) のインストールと拡張機能設定 (ESLint, Prettier)。
2.  **チュートリアル実践**:
    -   React公式の「三目並べ」チュートリアルを行う。
    -   Next.js公式の「Create a Next.js App」チュートリアルを行う。

### フェーズ2: 設計 (所要目安: 3日〜1週間)
AIがコードを書いてくれないため、事前の設計が手戻りを防ぐ鍵となります。

1.  **紙とペンでUIスケッチ**:
    -   どのような画面が必要か（シフト一覧、入力画面）を手書きで書き出す。
2.  **データ構造の検討**:
    -   「職員」「シフト」「業務」などのデータをJSON形式でどう持つかノートに書き出す。
3.  **コンポーネント分割**:
    -   UIスケッチを見ながら、「ボタン」「カレンダー」「ガントチャート」などの部品（コンポーネント）に丸をつけて名前をつける。

### フェーズ3: 実装 - プロトタイプ (所要目安: 1〜2週間)
まずは見た目と基本的な動きだけを作ります。

1.  **プロジェクト作成**:
    -   `npx create-next-app@latest` コマンドを実行。
2.  **静的な画面作成**:
    -   Tailwind CSSを使って、データが入っていない「見た目だけ」の画面（モックアップ）を作る。
    -   この段階ではロジック（JavaScript）は書かず、HTML(JSX)とCSSクラスのみに集中する。

### フェーズ4: 実装 - ロジック構築 (所要目安: 2〜3週間)
ここが最大の難所です。公式ドキュメントを読みながら進めます。

1.  **ステート管理の実装**:
    -   `useState` を使って、ボタンを押したら数字が増える等の単純な動きを作る。
    -   `Context API` を導入し、職員データなどをアプリ全体で共有できるようにする。
2.  **機能の実装**:
    -   **シフト入力**: プルダウンを変えたらStateが更新されるようにする。
    -   **ガントチャート**: 配列データを`map`関数で回して、`div`要素を並べて表示するロジックを書く。
    -   **計算ロジック**: `reduce`関数などを使って、シフトの回数を集計する関数を書く。
3.  **デバッグ**:
    -   `console.log` を活用して、変数の値が意図通りか確認しながらバグを潰す。

### フェーズ5: 仕上げとデプロイ (所要目安: 3日)
1.  **データ保存**:
    -   `useEffect` を使い、Stateが変化したら `localStorage` に保存する処理を追加する。
2.  **ビルド確認**:
    -   `npm run build` を実行し、エラーが出ないか確認する。
3.  **公開**:
    -   Vercelのアカウントを作り、GitHubリポジトリと連携してWeb上に公開する。

## 3. 成功のためのアドバイス

-   **「写経」をする**: チュートリアルのコードをコピー＆ペーストせず、自分で一行ずつタイプして構造を理解する。
-   **小さく作る**: 最初から高機能なガントチャートを作ろうとせず、まずは「文字だけでリスト表示」するなど、簡単なところから動くものを作る。
-   **エラーを読む**: エラーメッセージは答えです。英語でも翻訳ツールを使って内容を理解する習慣をつける。
